# عيادات المشفى — النسخة الاحترافية النهائية

نظام حجوزات ومتابعة دور احترافي لإدارة العيادات، يعمل فارغًا تمامًا بدون بيانات تجريبية، وجاهز للتشغيل المحلي أو النشر على GitHub Pages / Firebase.

## أهم التطويرات

- النظام يبدأ فارغًا بالكامل بدون أسماء أطباء أو مرضى أو حجوزات.
- إدارة غرف مستقلة: إضافة، تعديل، حذف، إيقاف/تفعيل.
- إدارة أطباء متقدمة: إضافة، تعديل، حذف، إيقاف/تفعيل، ربط بالغرف.
- لكل طبيب قائمة انتظار مستقلة ورقم دور مستقل.

- جدول عمل أسبوعي لكل طبيب: تحديد الأيام المتاحة ومن الساعة إلى الساعة، مع حد أقصى اختياري للحجوزات.
- الحجز أصبح مرتبطًا بيوم وتاريخ وساعة داخل مواعيد عمل الطبيب فقط.
- فلترة مركزية بتاريخ الحجز من الشريط العلوي؛ جميع الشاشات تعرض حالات اليوم المحدد فقط.
- أرقام الدور مستقلة لكل طبيب ولكل يوم حجز، ولا تختلط بين الأيام.
- شاشة الطبيب، شاشة الانتظار، الموبايل، والتحليلات تعمل حسب اليوم المحدد.
- لكل طبيب أكواد مرضى خاصة به مثل `DR1-0001`، ولا تختلط أكواد المرضى بين الأطباء.
- إدارة المرضى: بحث، تعديل، حذف، وربط كل مريض بطبيبه وكوده الخاص.
- إدارة الحجوزات: إضافة، تعديل، حذف، تغيير حالة، تأجيل، تخطي، إرجاع، ونداء التالي.
- طباعة تذكرة الدور من جدول الحجوزات.
- إرسال رسالة واتساب جاهزة من جدول الحجوزات مع متغيرات قابلة للتخصيص.
- شاشة طبيب احترافية لبدء الكشف، إنهائه، وطلب التالي.
- شاشة انتظار احترافية تعرض الرقم الحالي والأرقام القادمة حسب الطبيب المحدد.
- رابط متابعة للمريض بالهاتف أو الكود.
- لوحة تحليلات تشغيلية حسب الطبيب والغرفة والحالة وساعات المواعيد لليوم المحدد.
- إعدادات العيادة والفرع ومواعيد العمل ورسالة واتساب الافتراضية.
- تصدير واستيراد نسخة احتياطية JSON.
- مسح كل البيانات من الإعدادات.
- واجهة عربية RTL متجاوبة مع الموبايل والتابلت والشاشات الكبيرة.
- PWA جاهزة للتثبيت، وCapacitor جاهز لتطوير تطبيق Android/iOS.

## التشغيل

```bash
npm install
npm run dev
```

## بناء نسخة النشر

```bash
npm run build
```

سيتم إنشاء نسخة النشر داخل مجلد `dist`.

## ملاحظات مهمة

- التخزين الحالي يعمل عبر Local Storage داخل المتصفح.
- ملفات Firebase موجودة وجاهزة للتطوير السحابي عند الحاجة.
- لا توجد أي بيانات تجريبية داخل النسخة النهائية.
